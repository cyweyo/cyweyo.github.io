기간: 2025.01 ~ 2025.03

기술 스택: React Native, TypeScript, Firebase FCM

유형: 사용자 맞춤형 스포츠 매칭 서비스 (팀 프로젝트)

역할

  • 프론트엔드 아키텍처 설계 및 구현
  • 이메일 인증 회원가입 구현
  • 매치 생성 및 참여 시스템 구현
  • FCM토큰을 활용한 알림 기능 구현
  • 사용자 프로필 및 매치 관리 기능 구현
  • 매치 참여자 수락/거절 메시지 알림 기능 구현

주요 개발 성과

  1. 맞춤형 매칭 시스템 구현
    • 사용자의 신체 정보(키, 몸무게), 포지션, 레벨을 고려한 맞춤형 매칭 기능 구현
    • React의 내장 상태 관리(useState, useEffect)를 활용한 효율적인 데이터 핸들링
    • Props drilling 최소화와 컴포넌트 구조 최적화로 외부 라이브러리 없이 깔끔한 상태 관리 달성
  2. 실시간 알림 시스템 개발
    • Firebase Cloud Messaging(FCM)을 활용한 푸시 알림 구현
    • 매치 생성, 수락/거절, 완료 상태에 따른 알림 처리 로직 개발
    • 앱 내 실시간 메시지 표시 시스템 구축
  3. 성능 최적화
    • 코드 분할과 지연 로딩을 통한 앱 초기 실행 시간 25% 단축
    • 이미지 캐싱 구현으로 데이터 사용량 40% 절감
    • 메모이제이션을 통한 불필요한 리렌더링 방지

기술적 도전과 해결 방법

문제: 사용자 계정의 신뢰성 확보와 스팸 계정 방지를 위한 안전한 회원가입 프로세스 필요
해결: 이메일 인증 코드 기반 회원가입 시스템 구현으로 실제 이메일 소유 여부 확인 절차 도입
성과: 스팸 계정 생성 방지, 보안 강화 및 신뢰할 수 있는 사용자 데이터베이스 구축
이메일 인증 회원가입 흐름:
이메일 인증 시스템 장점:
  • 사용자 신원 확인을 통한 서비스 신뢰성 향상
  • 이메일 주소 검증으로 중요 알림 전달 보장 및 연락 가능성 확보
  • 타이머 기반 인증 코드 만료로 보안 강화
  • 재시도 제한 및 인증 실패 처리로 무차별 대입 공격 방지
문제: 토큰 기반 인증 시스템에서 AccessToken 탈취 시 보안 취약점 발생 가능성
해결: AccessToken과 RefreshToken을 분리하여 이중 보안 체계 구축. AccessToken에 짧은 만료 시간(6시간)을 설정하고, RefreshToken으로 자동 갱신되는 구조 구현
성과: 토큰 탈취 시에도 피해 범위 최소화 및 보안성 강화. 사용자 경험을 해치지 않으면서도 안전한 인증 시스템 구축
토큰 재발급 흐름:
문제: 각 컴포넌트에서 중복되는 API 호출 로직과 토큰 관리로 인한 코드 중복 및 일관성 부재
해결: Axios 인스턴스를 활용한 HTTP 클라이언트 추상화 구현으로 API 서비스 레이어 구축
성과: 인터셉터를 통한 자동 토큰 갱신, 중앙화된 에러 처리, 그리고 API 호출 로직 재사용성 향상
API클라이언트 추상화 흐름:
API클라이언트 추상화 장점:
  • 인터셉터를 통한 요청/응답 전처리 및 후처리 자동화
  • 토큰 관리와 갱신 프로세스를 중앙에서 처리
  • 에러 처리 로직 표준화

기간: 2024.08.05 ~ 2024.09.20

기술 스택: Next.js, React, Typescript, Zustand, TailwindCSS, AWS EC2, NPM

유형: 다양한 UI 컴포넌트를 제공하는 라이브러리 (팀 프로젝트)

역할

  • 주요 UI 컴포넌트(Alert, Breadcrumb, Dropdown 등) 개발 및 최적화
  • 팀원 간 코드 리뷰 및 협업 진행
  • 코드의 재사용성, 유지보수성, 성능 최적화, 그리고 확장성 개선

  2. 실시간 렌더링 시스템
    • 코드 수정 시 실시간으로 컴포넌트를 렌더링하도록 시스템 구현
    • 개발자 경험(DX) 향상을 위한 Live Preview 기능 개발

기술적 도전과 해결 방법

문제: 다양한 환경과 디자인 시스템에 유연하게 적용 가능한 컴포넌트 설계의 어려움
해결: 합성 패턴(Composition Pattern)과 Props 확장성을 고려한 컴포넌트 API 설계로 유연성 확보
성과: 다양한 디자인 요구사항에 쉽게 적응 가능한 컴포넌트 라이브러리 구축으로 재사용성 극대화
컴포넌트 합성 패턴 예시:
문제: DropdownMenu 컴포넌트에서 Trigger(메뉴를 여는 버튼)와 Content(메뉴 내용)가 서로 다른 컴포넌트로 분리되어 있어 상태를 공유하기 어려움
해결: Zustand 상태 관리 라이브러리를 도입하여 Trigger와 Content 간의 상태를 중앙에서 관리하도록 구현, 복잡한 props drilling 없이 각 컴포넌트에서 직접 상태에 접근 가능하게 함
성과: 컴포넌트 간 상태 공유가 원활해지고, 불필요한 리렌더링 방지로 성능 향상, 코드 가독성과 유지보수성 개선
Zustand 상태 관리 패턴:
문제: 사용자가 UI 컴포넌트 라이브러리를 학습할 때, 문서만 읽어서는 실제 사용 방법을 이해하기 어렵고, 컴포넌트 사용법을 실험해보려면 개발 환경을 직접 세팅해야 하는 불편함이 있었습니다
해결: Babel Standalone과 react-simple-code-editor를 활용하여 웹사이트 내에서 컴포넌트 코드를 직접 수정하고 그 결과를 실시간으로 확인할 수 있는 대화형 시스템 개발. 변환 엔진이 JSX 코드를 자동으로 해석해 실시간 미리보기를 제공합니다
성과: 사용자가 코드를 직접 수정하고 즉시 결과를 확인하며 실험할 수 있어 학습 곡선이 크게 완화되었고, "수정해보며 배우는" 직관적인 경험을 통해 컴포넌트 라이브러리 도입 장벽을 효과적으로 낮췄습니다
