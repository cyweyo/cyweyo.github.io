🚀 프로젝트 소개

Linkrew

기간: 2025.04 ~ 2025.07

기술 스택: React, TypeScript, Zustand, Axios, React Query, TailwindCSS

유형: 웹페이지 저장 및 공유 서비스 (팀 프로젝트, FE 2 / BE 2)

웹사이트 링크를 페이지·폴더 단위로 정리하고, 각 링크가 어떤 콘텐츠인지 직관적으로 확인할 수 있으며,
정리된 링크들을 다른 사용자와 손쉽게 공유할 수 있는 웹 서비스입니다.

역할
  • 폴더/링크 CRUD 구현: React Query와 Zustand를 활용해 일관된 상태 관리와 효율적인 캐시 처리 구현
  • 실시간 알림 기능: EventSource 기반 SSE 직접 구현, CORS 이슈를 헤더 설정으로 해결
  • 인증 흐름 개선: 토큰 만료 시 자동 리이슈 처리, UX를 고려한 인증 유지 흐름 설계
  • 회원 기능: 책임 단위 분리로 사용자 관리의 일관성 및 유지보수성 향상
주요 개발 성과
  • 링크별 대표 이미지 제공으로 콘텐츠 인지력 향상 (YouTube 썸네일, 블로그 OG 이미지 등)
  • SSE 알림 시스템 구축: 직접 백엔드와 연결 및 파싱 처리로 실시간 알림 기능 완성
  • 웹 서비스 아키텍처 전반에 대한 이해도 향상: 확장성과 재사용성을 고려한 컴포넌트 설계 경험 축적

🚧 Trouble Shooting

🛠️
문제
실시간 알림 기능을 구현하던 중, 알림 연결이 되지 않는 문제가 발생
💡
해결
브라우저의 EventSource 객체가 프록시를 통하지 않고 직접 서버에 요청을 보내는 구조임을 파악했습니다. 프론트 개발 서버와 백엔드 간 CORS 정책 충돌이 발생한 것이 원인이었고, 이에 따라 알림 요청을 프론트 프록시가 아닌 실제 백엔드 도메인으로 연결하고, 서버 측에서 Access-Control-Allow-Origin 및 관련 헤더를 설정하도록 백엔드와 협의하여 문제를 해결했습니다.
✅
성과
실시간 알림이 안정적으로 작동했고, 사용자 액션에 따른 UI 반영도 자연스럽게 구현됨
🛠️
문제
대부분의 API 요청은 정상 동작했지만, PATCH 요청만 CORS 오류로 실패함
💡
해결
브라우저가 PATCH 요청과 같이 민감한 HTTP 메서드를 사용할 경우, 사전 확인 요청(Preflight, OPTIONS)을 자동으로 보내는 구조임을 확인했습니다. 이 요청에 대해 백엔드가 CORS 허용 응답 헤더(Access-Control-Allow-Methods, Access-Control-Allow-Headers)를 반환하지 않아 오류가 발생한 것이었습니다. 문제 원인을 명확히 분석한 뒤, 백엔드에 필요한 헤더 값을 전달하고, 서버 코드를 수정하여 CORS 에러를 해결했습니다.
✅
성과
CORS 관련 이슈 해결로 PATCH 요청 안정화, 다양한 HTTP 메서드 지원 기반 구축
🛠️
문제
로그인 상태가 일정 시간 후 만료되면서, 사용자 경험이 끊기는 문제
💡
해결
AccessToken이 만료된 경우, 사용자는 로그인을 다시 해야 하는 불편이 있었습니다. 이를 해결하기 위해 RefreshToken을 이용한 자동 토큰 갱신 흐름을 설계했고, 토큰이 만료될 경우 전용 리이슈 페이지로 유도한 뒤, 새 토큰을 발급받아 자동으로 다시 로그인 상태로 전환되도록 구현했습니다. 이 과정을 사용자에게 노출되지 않게 구성하여 UX를 유지했습니다.
✅
성과
로그아웃 없이 인증을 유지할 수 있어, 사용자 입장에서 끊김 없는 경험을 제공함

문제: DropdownMenu 컴포넌트에서 Trigger(메뉴를 여는 버튼)와 Content(메뉴 내용)가 서로 다른 컴포넌트로 분리되어 있어 상태를 공유하기 어려움
